# Summarize Chat Skill

**Назначение:** Суммаризация переписки или обсуждения в структурированном виде.

## Входные данные

- `messages_json` — массив сообщений с полями [role, content, timestamp]
- `chat_name` — название чата/собеседник (опционально)

## Выходные данные

Markdown текст со структурированным саммари:

```markdown
📧 Саммари [название чата]

🔑 КЛЮЧЕВЫЕ ТЕМЫ
• Тема 1
• Тема 2

✅ РЕШЕНИЯ
• Что решили 1
• Что решили 2

❓ ТРЕБУЕТ ОТВЕТА
• Вопрос 1
• Вопрос 2

📝 ACTION ITEMS
• Кто: Что сделать (когда)
• Кто: Что сделать (когда)

💡 ВАЖНЫЕ МОМЕНТЫ
• Момент 1
• Момент 2
```

## Структура саммари

### 1. Ключевые темы
О чём говорили. Максимум 5 тем.

**Примеры:**
- Обсуждение нового проекта
- Планирование встречи
- Технические вопросы по реализации
- Feedback по презентации

### 2. Решения
Что конкретно решили/согласовали.

**Примеры:**
- Встреча назначена на среду в 14:00
- Бюджет проекта: 500k рублей
- Используем Notion для задач
- Deadline: конец месяца

**Если решений нет:**
```
✅ РЕШЕНИЯ
• Пока нет конкретных решений
```

### 3. Требует ответа
Вопросы, которые остались без ответа.

**Примеры:**
- Какой дизайнер будет работать?
- Когда можно посмотреть demo?
- Кто готовит документацию?

**Если все вопросы решены:**
```
❓ ТРЕБУЕТ ОТВЕТА
• Всё обсуждено
```

### 4. Action Items
Конкретные задачи с исполнителем и сроком.

**Формат:**
- **[Кто]**: [Что сделать] ([Когда])

**Примеры:**
- Борис: Подготовить презентацию (до пятницы)
- Алексей: Отправить договор (завтра)
- Команда: Провести ревью кода (среда)

**Если задач нет:**
```
📝 ACTION ITEMS
• Нет конкретных задач
```

### 5. Важные моменты
Дополнительная информация, факты, детали.

**Примеры:**
- Клиент хочет видеть прогресс еженедельно
- Используем стек: Next.js + TypeScript
- Deadline жёсткий, переноса не будет

## Алгоритм суммаризации

### 1. Анализ сообщений

Прочитай все сообщения и определи:
- **Темы** — о чём речь (группируй похожие)
- **Решения** — что согласовали, утвердили
- **Вопросы** — что спрашивали
- **Ответы** — что ответили
- **Задачи** — кто что должен сделать

### 2. Группировка

Сгруппируй похожие темы:
```
"Давайте встретимся" + "Во вторник подойдёт?" + "Ок, во вторник"
→ Тема: Планирование встречи
→ Решение: Встреча во вторник
```

### 3. Извлечение action items

Ищи паттерны:
- "Я сделаю X"
- "Ты можешь Y?"
- "Нужно Z до [дата]"
- "Пусть [имя] сделает"

### 4. Определение важности

**Важные моменты** — это:
- Упоминания о сроках
- Бюджеты, цифры
- Риски или проблемы
- Особые требования

## Примеры

### Пример 1: Деловая переписка
```
Input:
  messages: [
    {role: "user", content: "Алексей, давай обсудим новый проект"},
    {role: "other", content: "Отлично! Когда удобно?"},
    {role: "user", content: "Завтра в 14:00?"},
    {role: "other", content: "Подходит. Офис или Zoom?"},
    {role: "user", content: "Давай Zoom"},
    {role: "other", content: "Ок, жду ссылку"},
    {role: "user", content: "Отправлю завтра утром"}
  ]
  chat_name: "Алексей"

Output:
  📧 Саммари переписки с Алексеем

  🔑 КЛЮЧЕВЫЕ ТЕМЫ
  • Планирование встречи по новому проекту

  ✅ РЕШЕНИЯ
  • Встреча назначена на завтра в 14:00
  • Формат: Zoom

  ❓ ТРЕБУЕТ ОТВЕТА
  • Всё обсуждено

  📝 ACTION ITEMS
  • Борис: Отправить ссылку на Zoom (завтра утром)

  💡 ВАЖНЫЕ МОМЕНТЫ
  • Тема: Новый проект
```

```
Input:

  • Всё обсуждено

  📝 ACTION ITEMS
  • Собеседник: Настроить Supabase (до пятницы)

  💡 ВАЖНЫЕ МОМЕНТЫ
  • Deadline: пятница
```

## Краткость

**Правила краткости:**
- Максимум 5 ключевых тем
- Каждый пункт — 1 строка (максимум 2)
- Убирай детали второго уровня
- Фокус на действиях и решениях

## Тон

- **Нейтральный** — без эмоциональной окраски
- **Фактический** — только факты, не домыслы
- **Структурированный** — чёткие разделы
- **Actionable** — акцент на том, что делать дальше

## Обработка сложных случаев

### Если переписка очень длинная (100+ сообщений)
```
Сгруппируй по темам, не перечисляй каждое сообщение
```

### Если много off-topic
```
Игнорируй small talk, фокус на рабочих вопросах
```

### Если нет чётких решений
```
❓ ТРЕБУЕТ ОТВЕТА
• Нужно принять решение по [тема]
• Требуется обсуждение [тема]
```

## Важно

- Всегда указывай дедлайны если упоминались
- Не додумывай — только то, что было в переписке
- Action items должны быть конкретными (кто, что, когда)
- Если саммари длиннее 15 строк — сокращай
