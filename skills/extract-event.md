# Extract Event Skill

**Назначение:** Извлечение деталей события для создания в календаре.

## Входные данные

- `user_message` — сообщение с описанием события
- `current_datetime` — текущие дата и время

## Выходные данные

JSON объект:
```json
{
  "title": "Название события",
  "start_time": "ISO datetime",
  "end_time": "ISO datetime",
  "location": "Место или null",
  "description": "Описание или null",
  "attendees": ["email1@example.com"] или [],
  "recurrence": "RRULE или null"
}
```

## Извлечение полей

### Title (название)
Краткое название события (до 100 символов).

**Правила:**
- Убирай лишние слова ("встреча с", "созвон с" → имя)
- Сохраняй ключевую информацию

**Примеры:**
- "Встреча с Алексеем" → "Встреча с Алексеем"
- "Планерка по проекту X" → "Планерка: Проект X"
- "Созвон с клиентом" → "Созвон с клиентом"

### Start Time (начало)
ISO datetime когда начинается событие.

**Поддерживаемые форматы:**
- "сегодня в 15:00" → сегодня 15:00
- "завтра в 10 утра" → завтра 10:00
- "в пятницу в 14:00" → ближайшая пятница 14:00
- "через 2 часа" → текущее время + 2 часа
- "в 3 часа дня" → сегодня 15:00

**Время по умолчанию:**
- Если не указано и упомянуто "утром" → 10:00
- Если "днём" → 14:00
- Если "вечером" → 18:00
- Если вообще не указано → следующий полный час

### End Time (окончание)
ISO datetime когда заканчивается событие.

**Правила:**
- Если указана длительность: start_time + длительность
- Если не указано: start_time + 1 час (по умолчанию)

**Длительность:**
- "на полчаса" → +30 минут
- "на час" → +60 минут
- "на 2 часа" → +120 минут

### Location (место)
Где происходит событие.

**Примеры:**
- "в офисе" → "Офис"
- "Zoom" → "Zoom"
- "Кафе на Тверской" → "Кафе на Тверской"
- "переговорная 2" → "Переговорная 2"

**Если не указано:** null

### Description (описание)
Дополнительная информация о событии.

**Что включать:**
- Цель встречи
- Темы для обсуждения
- Подготовка

**Примеры:**
- "Обсуждение нового проекта" → description
- "Подготовить презентацию" → description

**Если не указано:** null

### Attendees (участники)
Список email адресов участников.

**Извлечение:**
- "с Алексеем" → попытаться найти email в контактах (или null)
- "alexey@example.com" → ["alexey@example.com"]
- "команда" → [] (не конкретные люди)

**Если не указано:** []

### Recurrence (повторение)
RRULE для повторяющихся событий.

**Поддерживаемые:**
- "каждый день" → FREQ=DAILY
- "каждую неделю" → FREQ=WEEKLY
- "каждый понедельник" → FREQ=WEEKLY;BYDAY=MO
- "раз в месяц" → FREQ=MONTHLY

**Если не указано:** null

## Обработка времени

### Относительное время
- "через час" → current_time + 1 час
- "через 2 дня в 10:00" → current_date + 2 дня, 10:00

### Неоднозначное время
- "в 3 часа" → определи по контексту:
  - Если сейчас 10:00 → 15:00 (день)
  - Если упомянуто "утра" → 03:00 (ночь)
  - По умолчанию → 15:00

### Дни недели
- "в понедельник" → ближайший понедельник
- "в следующий понедельник" → понедельник следующей недели

## Валидация

Проверь перед возвратом:
- Title не пустой
- Start time в будущем (не в прошлом)
- End time > start time
- ISO datetime в правильном формате
- RRULE синтаксически корректен (если указан)

## Умные предположения

### Если время не указано:
```
"Встреча с клиентом завтра"
→ Спросить: "В какое время?" или предложить: "14:00 подойдёт?"
```

### Если длительность неясна:
```
"Планерка" → обычно 30 мин - 1 час
"Воркшоп" → обычно 2-3 часа
"Созвон" → обычно 30 мин - 1 час
```

### Типичные места по типу события:
```
"Планерка" → часто Zoom/офис
"Обед" → ресторан/кафе
"Спортзал" → спортзал (очевидно)
```

## Важно

- Если дата/время неоднозначны → спроси пользователя
- Для важных встреч автоматически предлагай напоминание за 15 минут
- Сохраняй оригинальный смысл
- Часовой пояс: Europe/Moscow (по умолчанию)
