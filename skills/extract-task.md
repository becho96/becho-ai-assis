# Extract Task Skill

**Назначение:** Извлечение деталей задачи из естественного языка для создания в Notion Tasks.

## Входные данные

- `user_message` — сообщение с описанием задачи
- `current_datetime` — текущие дата и время для правильного парсинга

## Выходные данные

JSON объект:
```json
{
  "title": "Название задачи (краткое)",
  "priority": "high | medium | low",
  "due_date": "ISO datetime или null",
  "reminder": "ISO datetime или null",
  "notes": "Дополнительная информация или null",
  "project": "Название проекта или null"
}
```

## Извлечение полей

### Title (название)
Краткое, ёмкое название задачи (до 100 символов).

**Правила:**
- Начинай с глагола (Подготовить, Отправить, Позвонить)
- Убирай лишние слова ("нужно", "надо", "хочу")
- Сохраняй суть

**Примеры:**
- "Нужно подготовить презентацию" → "Подготовить презентацию"
- "Не забыть позвонить юристу" → "Позвонить юристу"
- "Хочу отправить отчёт клиенту" → "Отправить отчёт клиенту"

### Priority (приоритет)
Определяй автоматически по ключевым словам и контексту.

**HIGH (высокий):**
- Ключевые слова: срочно, важно, asap, критично, немедленно, urgent
- Дедлайн сегодня или завтра
- Слова "необходимо", "обязательно"

**MEDIUM (средний):**
- Дедлайн на этой неделе
- Нет явных указателей срочности
- По умолчанию, если не определён

**LOW (низкий):**
- Ключевые слова: когда-нибудь, если будет время, не срочно
- Дедлайн через 2+ недели или не указан
- "Было бы хорошо"

### Due Date (дедлайн)
Парси естественный язык в ISO datetime.

**Поддерживаемые форматы:**
- "сегодня" → текущий день, 23:59
- "завтра" → следующий день, 23:59
- "послезавтра" → через 2 дня, 23:59
- "в пятницу" → ближайшая пятница, 23:59
- "через неделю" → +7 дней, 23:59
- "до пятницы" → пятница, 23:59
- "к концу недели" → ближайшее воскресенье, 23:59
- "в конце месяца" → последний день месяца, 23:59
- "20 февраля" → указанная дата, 23:59

**Если не указано:** null

### Reminder (напоминание)
Когда напомнить о задаче.

**Автоматические правила:**
- Если есть "напомни" → извлечь время из сообщения
- Если дедлайн завтра и важно → напомнить сегодня вечером (18:00)
- Если дедлайн через неделю → напомнить за 1 день

**Поддерживаемые форматы:**
- "напомни завтра в 10" → завтра 10:00
- "напомни за день" → дедлайн минус 1 день
- "напомни утром" → следующий день 09:00

**Если не указано:** null (или автоматически для high priority)

### Notes (заметки)
Дополнительная информация, которая не поместилась в title.

**Примеры:**
- "Подготовить презентацию про MCP серверы для клиента"
  - Title: "Подготовить презентацию"
  - Notes: "Про MCP серверы для клиента"

- "Позвонить юристу по поводу договора"
  - Title: "Позвонить юристу"
  - Notes: "По поводу договора"

### Project (проект)
Если упоминается принадлежность к проекту.

**Примеры:**
- "Задача для проекта Becho" → project: "Becho"
- "По проекту X нужно..." → project: "X"

**Если не указано:** null

## Примеры извлечения

### Пример 1: Простая задача
```
Input:
  user_message: "Подготовить презентацию"
  current_datetime: "2026-02-20T14:00:00"

Output:
  {
    "title": "Подготовить презентацию",
    "priority": "medium",
    "due_date": null,
    "reminder": null,
    "notes": null,
    "project": null
  }
```

    "reminder": "2026-02-21T09:00:00",

  }

## Обработка сложных случаев

### Множественные задачи
```
Input: "Подготовить презентацию и отправить отчёт"
→ Создать 2 отдельные задачи
```

### Относительные даты
```
"через 3 дня" → текущая дата + 3 дня
"на следующей неделе" → начало следующей недели (понедельник)
```

### Неоднозначное время
```
"в 3 часа" → контекст:
  - Если сейчас утро → 15:00
  - Если упомянуто "утра" → 03:00
  - По умолчанию → 15:00
```

### Повторяющиеся задачи
```
Input: "Каждый день проверять почту"
→ Добавить в notes: "Повторяется ежедневно"
→ В будущем можно будет добавить recurrence rule
```

## Валидация

Проверь перед возвратом:
- Title не пустой и не длиннее 100 символов
- Priority один из: high, medium, low
- Due date в будущем (не в прошлом)
- Reminder раньше due_date (если оба указаны)
- ISO datetime в правильном формате

## Важно

- Сохраняй оригинальный смысл, не додумывай лишнего
- Если дата неоднозначна → спроси пользователя
- Приоритет определяй консервативно (по умолчанию medium)
- Для срочных задач автоматически создавай reminder
