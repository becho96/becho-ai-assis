# Draft Message Skill

**Назначение:** Помощь в составлении сообщений с правильным тоном и стилем.

## Входные данные

- `conversation_context` — контекст переписки (история сообщений)
- `recipient_name` — имя получателя
- `purpose` — цель сообщения (договориться, уточнить, поблагодарить, etc)
- `tone` — тон (formal/informal/friendly/business)
- `language` — язык (ru/en)
- `key_points` — ключевые пункты для включения (опционально)

## Выходные данные

Markdown с черновиком сообщения:
```markdown
✍️ Черновик сообщения:

---
[Текст сообщения]
---

Длина: [количество слов]
Тон: [описание тона]
```

## Тоны сообщений

### FORMAL (формальный, деловой)
**Когда использовать:**
- Первое обращение к незнакомому человеку
- Официальная переписка
- Переписка с руководством, клиентами

**Характеристики:**
- Обращение на "Вы"
- Полные предложения
- Официальные приветствия (Здравствуйте, Уважаемый)
- Подпись (С уважением, Best regards)
- Без emoji
- Структурированный текст

**Пример:**
```
Здравствуйте, Алексей!

Благодарю за оперативный ответ. По обсужда

емому вопросу предлагаю следующее:
1. [Пункт 1]
2. [Пункт 2]

Буду рад услышать ваше мнение.

С уважением,
Борис
```

### INFORMAL (неформальный, рабочий)
**Когда использовать:**
- Коллеги
- Регулярная переписка
- Внутренние чаты

**Характеристики:**
- Можно на "ты" (если установлен контакт)
- Короче и проще
- Приветствие: Привет, Здравствуй
- Emoji уместны (умеренно)
- Разговорная речь

**Пример:**
```
Привет, Алексей! 👋

Спасибо за фидбек. По проекту предлагаю:
• [Пункт 1]
• [Пункт 2]

Как смотришь?
```

### FRIENDLY (дружеский)
**Когда использовать:**
- Друзья
- Близкие знакомые
- Непринуждённое общение

**Характеристики:**
- Обязательно "ты"
- Очень короткие предложения
- Emoji приветствуются
- Разговорные фразы
- Без официоза

**Пример:**
```
Привет! 😊

Всё круто, давай так и сделаем! 🚀

По пунктам:
1. [Пункт 1] ✅
2. [Пункт 2] 👍

Жду твоих мыслей!
```

### BUSINESS (строго деловой)
**Когда использовать:**
- Контракты, договорённости
- Официальные запросы
- B2B переписка

**Характеристики:**
- Максимально официально
- Чёткая структура
- Нумерованные списки
- Даты и сроки
- Без emoji вообще
- Юридически точные формулировки

**Пример:**
```
Уважаемый Алексей Иванович,

В продолжение нашего разговора от 20.02.2026, направляю на рассмотрение следующие предложения:

1. Срок реализации: до 31.03.2026
2. Бюджет: 500 000 (Пятьсот тысяч) рублей
3. Условия оплаты: 50% аванс, 50% по завершении

Прошу подтвердить согласие до 22.02.2026.

С уважением,
Борис Петров
Должность, Компания
```

## Цели сообщений

### Договориться о встрече
```
Структура:
1. Приветствие
2. Предложение встретиться
3. Варианты времени (2-3)
4. Формат (офис/zoom/кафе)
5. Подтверждение
```

**Пример:**
```
Привет, Алексей!

Давай обсудим новый проект. Предлагаю встретиться на этой неделе.

Мне подходит:
• Среда, 14:00
• Четверг, 10:00
• Пятница, 16:00

Zoom или в офисе — как удобнее?
```

### Уточнить информацию
```
Структура:
1. Контекст (о чём речь)
2. Конкретные вопросы
3. Зачем нужна информация
4. Когда ждёшь ответ
```

**Пример:**
```
Алексей, по проекту пара уточнений:

1. Какой дедлайн по макетам?
2. Кто отвечает за бэкенд?
3. Нужен ли отдельный QA?

Нужно для планирования спринта.
Можешь ответить до среды?
```

### Поблагодарить
```
Структура:
1. Благодарность
2. За что конкретно
3. Как это помогло
4. (Опционально) Что дальше
```

**Пример:**
```
Алексей, спасибо за оперативную помощь! 🙏

Твои комментарии по коду очень помогли —
исправил всё за полчаса.

Завтра отправлю финальную версию на ревью.
```

### Отказать/Отклонить
```
Структура:
1. Благодарность за предложение
2. Вежливый отказ
3. Причина (опционально)
4. Альтернатива (если есть)
```

**Пример (формальный):**
```
Здравствуйте, Алексей!

Благодарю за предложение. К сожалению, в текущий момент не могу взять проект — полностью загружен до конца месяца.

Могу порекомендовать коллегу, если интересно.

С уважением,
Борис
```

### Напомнить
```
Структура:
1. Мягкое напоминание
2. О чём речь (контекст)
3. Когда ждали
4. Когда нужно сейчас
```

**Пример:**
```
Привет, Алексей!

Напоминаю про договор, который обсуждали на прошлой неделе.

Планировали подписать до пятницы.
Когда удобно финализировать?
```

## Языки

### Русский (ru)
- Дефолтный язык
- Используй если не указано иное
- Учитывай русские формальности (Вы/ты)

### English (en)
- Если указан явно
- Если recipient англоязычный
- Формальность: Dear/Hi/Hey
- Структура аналогична русской

**Formal English:**
```
Dear Alex,

Thank you for your prompt response.
Regarding the discussed matter, I propose the following:

1. [Point 1]
2. [Point 2]

Looking forward to your feedback.

Best regards,
Boris
```

**Informal English:**
```
Hi Alex! 👋

Thanks for the feedback. Here's what I'm thinking:
• [Point 1]
• [Point 2]

What do you think?
```

## Структурирование

### Короткие сообщения (до 3 строк)
```
Без структуры, просто текст
```

### Средние сообщения (3-10 строк)
```
• Bullet points
• Для списков
```

### Длинные сообщения (10+ строк)
```
1. Нумерованные списки
2. Для последовательных пунктов
3. Или когда порядок важен
```

## Примеры с контекстом

### Пример 1: Первое сообщение клиенту
```
Input:
  recipient_name: "Алексей Иванович"
  purpose: "Представиться и предложить встречу"
  tone: "formal"
  language: "ru"

Output:
  Здравствуйте, Алексей Иванович!

  Меня зовут Борис, я занимаюсь разработкой
  AI-решений. Коллеги рекомендовали обратиться
  к вам по вопросу автоматизации процессов.

  Было бы интересно обсудить возможное
  сотрудничество. Можем созвониться на этой неделе?

  Буду рад услышать ваше мнение.

  С уважением,
  Борис
```

### Пример 2: Сообщение коллеге
```
Input:
  conversation_context: [обсуждали проект]
  recipient_name: "Алексей"
  purpose: "Согласовать дедлайн"
  tone: "informal"
  language: "ru"

Output:
  Привет, Алексей!

  По проекту: давай дедлайн поставим на конец недели?
  У меня есть 2-3 дня запаса, если что.

  Подходит?
```

### Пример 3: Благодарность другу
```
Input:
  recipient_name: "Алексей"
  purpose: "Поблагодарить за помощь"
  tone: "friendly"
  language: "ru"

Output:
  Лёха, спасибо огромное! 🙏

  Ты меня прямо выручил с этим кодом.
  Без тебя бы до утра сидел 😅

  Должен тебе пиво! 🍺
```

## Валидация черновика

Проверь перед отправкой:
- ✅ Тон соответствует отношениям с получателем
- ✅ Все key_points включены
- ✅ Нет грамматических ошибок
- ✅ Длина адекватна (не слишком длинно/коротко)
- ✅ Призыв к действию ясен (если нужен ответ)

## Важно

- **Тон** — самое важное, неправильный тон может испортить отношения
- **Краткость** — чем короче, тем лучше (при сохранении смысла)
- **Ясность** — получатель должен понять, что от него требуется
- **Контекст** — учитывай историю переписки
- **Культура** — русская деловая культура отличается от западной
