# Classify Intent Skill

**Назначение:** Классификация намерения пользователя для правильной маршрутизации к специализированному агенту.

## Входные данные

- `user_message` — сообщение пользователя
- `recent_context` — последние 5-10 сообщений для контекста

## Выходные данные

JSON объект:
```json
{
  "intent": "NOTE | TASK | REMINDER | CALENDAR | SEARCH | COMMUNICATE | DIGEST | GENERAL",
  "confidence": 0.0-1.0,
  "entities": {
    "subject": "о чём речь",
    "urgency": "high | medium | low",
    "time_reference": "when mentioned (если есть)"
  }
}
```

## Типы намерений

### NOTE (Заметка)
Пользователь хочет сохранить информацию.

**Ключевые слова:**
- запиши, сохрани, заметка, note
- идея, мысль
- запомни

**Примеры:**
- "Запиши: встреча прошла отлично"
- "Сохрани эту идею"
- "Заметка про RAG"

### TASK (Задача)
Пользователь хочет создать задачу или to-do item.

**Ключевые слова:**
- задача, сделать, нужно, todo
- не забыть, выполнить
- срочно, важно

**Примеры:**
- "Подготовить презентацию до пятницы"
- "Нужно позвонить юристу"
- "Срочно: отправить отчёт"

### REMINDER (Напоминание)
Пользователь хочет установить напоминание.

**Ключевые слова:**
- напомни, напоминание, remind
- в [время], завтра в, через

**Примеры:**
- "Напомни завтра в 10 утра позвонить"
- "Напоминание: встреча в среду"
- "Напомни через час проверить почту"

### CALENDAR (Календарь)
Пользователь спрашивает про расписание или хочет создать событие.

**Ключевые слова:**
- что сегодня, расписание, schedule
- встреча, событие, calendar
- свободен ли я, занят ли

**Примеры:**
- "Что у меня сегодня?"
- "Встреча с Алексеем завтра в 3"
- "Свободен ли я в пятницу?"

### SEARCH (Поиск)
Пользователь ищет информацию.

**Ключевые слова:**
- найди, поиск, search
- что такое, как, explain
- информация про, расскажи о

**Примеры:**
- "Найди информацию про CalDAV"
- "Что такое RAG?"
- "Как реализовать MCP сервер?"

### COMMUNICATE (Коммуникация)
Пользователь хочет помощь с сообщениями.

**Ключевые слова:**
- напиши, составь письмо
- ответь, отправь
- что ответить, как сказать

**Примеры:**
- "Напиши письмо Алексею"
- "Что ответить на это сообщение?"
- "Помоги составить ответ"

### DIGEST (Дайджест)
Пользователь хочет сводку/саммари.

**Ключевые слова:**
- сводка, дайджест, summary
- что было, итоги
- план на день, расскажи про день

**Примеры:**
- "Дай сводку на сегодня"
- "Что было за неделю?"
- "План на завтра"

### GENERAL (Общее)
Обычный разговор, приветствие, вопрос.

**Ключевые слова:**
- привет, здравствуй, hi
- как дела, что нового
- спасибо, thanks

**Примеры:**
- "Привет!"
- "Как дела?"
- "Спасибо за помощь"

### DIALOGUE_ANALYSIS (Анализ диалогов)
Пользователь хочет проанализировать переписку в Telegram.

**Ключевые слова:**
- проанализируй переписку, анализ диалога, dialogue analysis
- что обсуждали с, что решили в чате
- сформируй задачи из переписки, extract tasks from chat
- покажи сводку диалога, summarize chat
- что было в переписке, conversation summary

**Примеры:**
- "Проанализируй переписку с @zadum_off за последние 7 дней"
- "Что обсуждали с командой на этой неделе?"
- "Сформируй список задач из диалога с клиентом"
- "Покажи сводку переписки с партнером"
- "Что решили в чате с @username?"

**Entities:**
```json
{
  "target_user": "@username или название чата",
  "time_range": "7 days, this week, 30 days",
  "analysis_type": "full | tasks_only | summary | risks"
}
```

## Логика классификации

1. **Проверь явные триггеры:**
   - "Запиши" → NOTE (высокая уверенность)
   - "Напомни" → REMINDER (высокая уверенность)
   - "Что сегодня" → CALENDAR (высокая уверенность)

2. **Проверь контекст:**
   - Если предыдущее сообщение было вопросом "что сохранить?" → NOTE
   - Если говорили о задаче и пришло "сделано" → TASK (обновление)

3. **Анализ временных отсылок:**
   - "завтра в 10" → скорее всего REMINDER или CALENDAR
   - "до пятницы" → скорее всего TASK

4. **Множественные намерения:**
   - "Запиши и напомни завтра" → оба NOTE и REMINDER
   - В таком случае главное намерение = первое упомянутое

5. **Неоднозначность:**
   - Если confidence < 0.7 → GENERAL + попросить уточнить

## Примеры классификации

### Пример 1: Простая заметка
```
Input:
  user_message: "Запиши: встреча с инвестором прошла отлично"
  recent_context: []

Output:
  {
    "intent": "NOTE",
    "confidence": 0.95,
    "entities": {
      "subject": "встреча с инвестором",
      "urgency": "medium"
    }
  }
```

### Пример 2: Задача с дедлайном
```
Input:
  user_message: "Подготовить презентацию до пятницы"
  recent_context: []

Output:
  {
    "intent": "TASK",
    "confidence": 0.9,
    "entities": {
      "subject": "подготовить презентацию",
      "urgency": "medium",
      "time_reference": "до пятницы"
    }
  }
```

### Пример 3: Напоминание
```
Input:
  user_message: "Напомни завтра в 10 позвонить юристу"
  recent_context: []

Output:
  {
    "intent": "REMINDER",
    "confidence": 0.95,
    "entities": {
      "subject": "позвонить юристу",
      "urgency": "medium",
      "time_reference": "завтра в 10"
    }
  }
```

### Пример 4: С контекстом
```
Input:
  user_message: "Да, сохрани"
  recent_context: [
    "User: Интересная идея про использование RAG",
    "Assistant: Хочешь сохранить в базу знаний?"
  ]

Output:
  {
    "intent": "NOTE",
    "confidence": 0.85,
    "entities": {
      "subject": "идея про RAG",
      "urgency": "low"
    }
  }
```

### Пример 5: Неоднозначность
```
Input:
  user_message: "презентация"
  recent_context: []

Output:
  {
    "intent": "GENERAL",
    "confidence": 0.3,
    "entities": {
      "subject": "презентация"
    }
  }

→ Orchestrator должен спросить: "Что с презентацией? Создать задачу, запомнить или найти информацию?"
```

## Обработка сложных случаев

### Множественные действия
```
Input: "Запиши встречу и напомни завтра"
Output: NOTE (основное) + создать REMINDER тоже
```

### Вопросы
```
Input: "Когда у меня встреча с Алексеем?"
Output: CALENDAR (поиск события)
```

### Команды обновления
```
Input: "Отметь задачу выполненной"
Output: TASK (обновление статуса)
```

## Важно

- Всегда учитывай контекст предыдущих сообщений
- Если уверенность < 0.7 → лучше уточнить у пользователя
- При множественных намерениях возвращай основное + пометку о дополнительных
- Временные отсылки (завтра, в пятницу) — важный сигнал для TASK/REMINDER/CALENDAR
