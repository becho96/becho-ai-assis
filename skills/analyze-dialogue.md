# Analyze Dialogue Skill

**Назначение:** Глубокий анализ переписки в Telegram с извлечением insights, задач, решений и рисков.

## Входные данные

- `messages` — массив сообщений из диалога (JSON)
- `analysis_type` — тип анализа (optional):
  - `full` — полный анализ (default)
  - `tasks_only` — только задачи
  - `summary` — краткое содержание
  - `risks` — выявление рисков
- `time_range` — временной диапазон (для контекста)

## Выходные данные

JSON объект:
```json
{
  "summary": "Краткое содержание диалога в 2-3 предложениях",
  "key_topics": [
    "Тема 1",
    "Тема 2"
  ],
  "decisions": [
    "Решение 1: описание",
    "Решение 2: описание"
  ],
  "action_items": [
    {
      "task": "Описание задачи",
      "assignee": "Имя или username",
      "due_date": "YYYY-MM-DD или null",
      "priority": "high | medium | low",
      "context": "Контекст из диалога"
    }
  ],
  "questions_unanswered": [
    "Открытый вопрос 1",
    "Открытый вопрос 2"
  ],
  "risks": [
    "Риск 1: описание",
    "Риск 2: описание"
  ],
  "next_steps": [
    "Следующий шаг 1",
    "Следующий шаг 2"
  ],
  "important_dates": [
    "2026-02-20: Дедлайн презентации",
    "2026-02-25: Встреча с клиентом"
  ]
}
```

## Алгоритм анализа

### Шаг 1: Прочитать все сообщения
- Изучить контекст разговора
- Определить участников
- Понять временные рамки

### Шаг 2: Выделить ключевые темы
- Какие основные темы обсуждались?
- Группировать похожие темы
- Выделить 3-5 главных тем

### Шаг 3: Найти принятые решения
- Искать фразы: "решили", "договорились", "утвердили", "согласовали"
- Конкретные решения, а не общие обсуждения
- Кто принял решение

### Шаг 4: Извлечь задачи (action items)
- Искать фразы: "нужно сделать", "я сделаю", "ты сделаешь", "надо"
- Определить assignee (кто должен выполнить)
- Извлечь дедлайн если есть ("до пятницы", "завтра", "к концу недели")
- Определить приоритет:
  - **high**: срочные задачи, критические
  - **medium**: обычные задачи с дедлайном
  - **low**: задачи без дедлайна, несрочные

### Шаг 5: Определить открытые вопросы
- Вопросы, которые остались без ответа
- Нерешенные проблемы
- То, что требует уточнения

### Шаг 6: Выявить риски
- Потенциальные проблемы
- Упоминания о сложностях
- Зависимости от внешних факторов
- Временные риски (tight deadlines)

### Шаг 7: Сформулировать next steps
- Что нужно сделать дальше
- Логическое продолжение обсуждения
- Следующие встречи или действия

### Шаг 8: Извлечь важные даты
- Дедлайны
- Встречи
- События
- Формат: "YYYY-MM-DD: Описание"

## Обработка сложных случаев

### Множество участников
- Определить кто что сказал
- Выделить assignee для каждой задачи
- Не терять контекст при смене говорящего

### Длинные диалоги (> 200 сообщений)
- Группировать по темам
- Фокусироваться на ключевых моментах
- Не упустить важные детали

### Неформальный стиль
- Распознавать задачи даже без явных указаний
  - "кстати надо бы это сделать" → task
  - "окей, я попробую" → task
- Понимать контекст и подтекст

### Смешанные языки (русский + английский)
- Анализировать оба языка
- Сохранять термины в оригинале

### Нет явных задач
- Извлечь неявные action items
- Определить next steps из контекста

## Важно

- **Фокус на действиях:** Задачи должны быть конкретными и выполнимыми
- **Контекст:** Всегда сохранять контекст задачи из диалога
- **Точность дат:** Если дата упомянута ("до пятницы") - вычислить конкретную дату
- **Приоритет:** Определять реалистично, не завышать
- **Открытые вопросы:** Выделять то, что требует уточнения
- **Риски:** Быть объективным, не преувеличивать
