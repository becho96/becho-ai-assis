## 🤖 Описание агентов

### 1. Orchestrator (Оркестратор)
**Роль:** Центральный агент-маршрутизатор

**Ответственность:**
- Получает сообщения от пользователя
- Классифицирует намерения (через скил classify-intent)
- Делегирует задачи специализированным агентам
- Сохраняет контекст разговора в SQLite
- Форматирует ответы для Telegram

**Инструменты:**
- `db_get_context` — история разговора
- `db_save_conversation` — сохранение сообщений
- `telegram_send_message` — отправка ответов

### 2. Note Taker (Заметки)
**Роль:** Управление базой знаний в Notion

**Ответственность:**
- Создание заметок в Knowledge Base
- Автоматическая категоризация (work/personal/learning/etc)
- Автоматические теги (important/idea/reference)
- Поиск заметок
- Обновление существующих заметок

**Инструменты:**
- `notion_create_page` — создание заметок
- `notion_search` — поиск заметок
- `notion_query_database` — фильтрация
- `notion_update_page` — обновление

### 3. Task Manager (Задачи)
**Роль:** Управление задачами и напоминаниями

**Ответственность:**
- Создание задач в Notion Tasks
- Настройка напоминаний в SQLite
- Просмотр активных задач
- Обновление статуса (todo → done)
- Автоматическая приоритизация

**Инструменты:**
- `notion_create_page` — создание задач
- `notion_query_database` — список задач
- `notion_update_page` — обновление статуса
- `db_save_reminder` — напоминания

**Скилы:**
- `extract-task` — извлечение деталей задачи

### 4. Scheduler (Календарь)
**Роль:** Управление расписанием и календарём

**Ответственность:**
- Просмотр расписания (день/неделя)
- Создание событий в календаре
- Проверка свободного времени
- Предложение удобных слотов

**Инструменты:**
- `calendar_list_events` — список событий
- `calendar_create_event` — создание событий
- `calendar_free_busy` — проверка занятости

**Скилы:**
- `extract-event` — извлечение деталей события

### 5. Researcher (Исследователь)
**Роль:** Поиск и анализ информации

**Ответственность:**
- Веб-поиск через Perplexity
- Поиск в базе знаний Notion
- Синтез информации из разных источников
- Сохранение находок в базу знаний

**Инструменты:**
- `perplexity_search` — умный поиск
- `notion_search` — поиск в Knowledge Base
- `notion_create_page` — сохранение находок

### 6. Communicator (Коммуникатор)
**Роль:** Помощь в переписке

**Ответственность:**
- Чтение и анализ переписок
- Составление черновиков сообщений
- Суммаризация обсуждений
- Отслеживание неотвеченных сообщений

**Инструменты:**
- `telegram_send_message` — отправка сообщений
- `notion_create_page` — сохранение важных переписок
- `db_get_context` — история общения

**Скилы:**
- `summarize-chat` — суммаризация переписок
- `draft-message` — составление сообщений

### 7. Daily Digest (Дайджесты)
**Роль:** Генерация сводок

**Ответственность:**
- Утренний дайджест (расписание, задачи, напоминания)
- Вечерний дайджест (итоги дня, план на завтра)
- Недельный обзор
- Сохранение в Daily Log

**Инструменты:**
- `notion_query_database` — задачи, заметки
- `calendar_list_events` — события
- `db_query` — напоминания
- `notion_create_page` — Daily Log

## 🎨 Скилы (Промпт-шаблоны)

### 1. Classify Intent
**Назначение:** Классификация намерения пользователя

**Типы намерений:**
- NOTE — создать заметку
- TASK — создать задачу
- REMINDER — установить напоминание
- CALENDAR — работа с расписанием
- SEARCH — поиск информации
- COMMUNICATE — помощь с сообщениями
- DIGEST — запрос дайджеста
- GENERAL — обычный разговор

**Выход:** JSON с intent, confidence, entities

### 2. Extract Task
**Назначение:** Извлечение деталей задачи из естественного языка

**Извлекает:**
- Title — название задачи
- Priority — high/medium/low (автоматически)
- Due Date — дедлайн (парсит "завтра", "в пятницу", etc)
- Reminder — когда напомнить
- Notes — доп. информация
- Project — проект

**Выход:** JSON с полями задачи

### 3. Extract Event
**Назначение:** Извлечение деталей события для календаря

**Извлекает:**
- Title — название события
- Start Time — начало (парсит естественный язык)
- End Time — окончание (или +1 час по умолчанию)
- Location — место
- Description — описание
- Attendees — участники
- Recurrence — RRULE для повторяющихся событий

**Выход:** JSON с полями события

### 4. Summarize Chat
**Назначение:** Суммаризация переписки

**Структура саммари:**
- Ключевые темы
- Решения
- Вопросы требующие ответа
- Action items (кто, что, когда)
- Важные моменты

**Выход:** Markdown с структурированным саммари

### 5. Draft Message
**Назначение:** Составление сообщений с правильным тоном

**Поддерживаемые тоны:**
- FORMAL — формальный, деловой (Вы, официально)
- INFORMAL — неформальный, рабочий (ты/Вы, проще)
- FRIENDLY — дружеский (ты, emoji, непринуждённо)
- BUSINESS — строго деловой (контракты, B2B)

**Цели:**
- Договориться о встрече
- Уточнить информацию
- Поблагодарить
- Отказать/Отклонить
- Напомнить

**Выход:** Markdown с черновиком сообщения

## 🔄 Взаимодействие агентов

```
Пользователь (Telegram)
    ↓
Orchestrator
    ├─ классифицирует намерение (classify-intent)
    ├─ получает контекст (db_get_context)
    └─ делегирует агенту:
        ├─ NOTE → Note Taker
        ├─ TASK → Task Manager
        ├─ CALENDAR → Scheduler
        ├─ SEARCH → Researcher
        ├─ COMMUNICATE → Communicator
        ├─ DIGEST → Daily Digest
        └─ GENERAL → отвечает сам
    ↓
Специализированный агент
    ├─ использует скилы (extract-*, draft-*, etc)
    ├─ вызывает MCP инструменты
    └─ возвращает результат
    ↓
Orchestrator
    ├─ форматирует ответ
    ├─ сохраняет в историю (db_save_conversation)
    └─ отправляет пользователю (telegram_send_message)
```

## 📝 Примеры использования

### Пример 1: Создание заметки
```
User: "Запиши: встреча с инвестором прошла отлично"
    ↓
Orchestrator: classify-intent → NOTE
    ↓
Note Taker:
  - Title: "Встреча с инвестором"
  - Category: "work"
  - Tags: ["important"]
  - notion_create_page()
    ↓
Response: "✅ Заметка создана в категории work
🔗 [Ссылка на Notion]"
```

### Пример 2: Создание задачи с напоминанием
```
User: "Напомни завтра в 10 утра позвонить юристу"
    ↓
Orchestrator: classify-intent → REMINDER + TASK
    ↓
Task Manager:
  - extract-task(message)
  - notion_create_page(tasks)
  - db_save_reminder(tomorrow 10:00)
    ↓
Response: "✅ Задача создана с напоминанием на завтра 10:00"
```

### Пример 3: Поиск информации
```
User: "Найди информацию про CalDAV protocol"
    ↓
Orchestrator: classify-intent → SEARCH
    ↓
Researcher:
  - notion_search("CalDAV") → проверяет базу
  - perplexity_search("CalDAV protocol") → ищет в интернете
  - синтезирует результаты
    ↓
Response: "[Информация с источниками]
💾 Сохранить в базу знаний?"
```

### Пример 4: Помощь с сообщением
```
User: "Напиши письмо Алексею про перенос встречи"
    ↓
Orchestrator: classify-intent → COMMUNICATE
    ↓
Communicator:
  - draft-message(
      recipient: "Алексей",
      purpose: "перенос встречи",
      tone: "informal"
    )
    ↓
Response: "✍️ Черновик:
---
Привет, Алексей!

К сожалению, нужно перенести нашу встречу.
Подойдёт среда в 14:00?
---
Отправить?"
```
